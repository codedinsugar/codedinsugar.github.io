httPyze - a Python model for secretProject

December 23, 2022

  • blog
  • python

Everything has beauty, but not everyone sees it.

-Confucius

Python how I love thee, let me count the ways

In a recent social engagement, the topic of discussion was Python. Somewhere in the middle of the conversation I said something along the lines of “I love Python, been reading it for years, wish I could write more of it everyday”. Then it dawned on me, why don’t I write Python everyday? As a Senior SRE, I have to know about a lot of different technologies and how they provide solutions to problems no matter how simple or complex. Granted I definitely read code everyday but I don’t always get to write code everyday. Let’s change that.

httPyze meets secretProject

My original plan for secretProject was to build everything entirely in Go for the simple reason of wanting to learn Go. But why should I limit it to just one language? That’s not realistic at all. Let’s change that.

To simulate what I do for work everyday the best approach here is to build some microservices in Go and some in Python. This way I can develop an observability palette of different flavors.

Some initial menu items that come to mind:

  • A simple API using from http.server import HTTPServer, BaseHTTPRequestHandler
  • Using podman to build the image and push it to a private DockerHub registry
  • Deploy httPyze as a Kubernetes Deployment + nodePort Service

Let the fun begin!
